# 产品管理功能 — 设计规格

**日期**: 2026-06-10
**状态**: 待审核

---

## 1. 概述

左侧侧边栏新增「📦 产品管理」，管理产品/包/链接的层级关系。支持增删改查，一键导入导出 Excel。

## 2. 数据模型

### 层级关系

```
产品（产品/群名）            ← 一对一 →  KPI
      │                      ← 一对一 →  地区
      │
      └── 包列表（多个）
           ├── 系列名 + 包名 + 链接    （一对一）
           ├── 系列名 + 包名 + 链接
           └── ...
```

- 产品 ↔ KPI ↔ 地区：一对一，产品级别属性
- 每个包存储：系列名、包名、链接（三个一对一）
- 一个产品下有多个包

### 数据库（SQLite，复用 `temp/youtube.db`）

**products 表**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增 |
| product_name | TEXT | 产品/群名 |
| kpi | TEXT | KPI |
| region | TEXT | 地区 |
| is_paused | INTEGER | 0=正常, 1=暂停 |
| created_at | TEXT | |

**packages 表**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增 |
| product_id | INTEGER FK | 关联产品 |
| series_name | TEXT | 系列名 |
| package_name | TEXT | 包名（com.xxx.xxx） |
| url | TEXT | Google Play 链接 |
| is_paused | INTEGER | 0=正常, 1=暂停 |
| created_at | TEXT | |

## 4. 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/products/list?search=&region=&page=&size=` | 产品列表 |
| POST | `/api/products/create` | 新增产品+包 |
| PUT | `/api/products/<id>` | 编辑产品信息 |
| DELETE | `/api/products/<id>` | 删除产品及关联包 |
| POST | `/api/products/<id>/packages` | 给产品加包 |
| DELETE | `/api/products/packages/<pid>` | 删单个包 |
| GET | `/api/products/export` | 导出 Excel |
| POST | `/api/products/import-excel` | Excel 导入 |
| POST | `/api/products/import-text` | 复制文本导入 |

## 6. 脏数据解析规则

五种脏数据格式及提取规则：

### 通用规则
- **包名**：始终从链接的 `id=` 参数提取，不依赖文本中的包名
- **链接**：正则 `https?://play\.google\.com/store/apps/details\?id=[\w.]+`

### 类型1：多行结构化（"📱" 前缀）
```
📱FF345-APK-24 三方回传：AD
📱APP包名：com.foldhueabc.logicjklmno
📱APP链接：https://play.google.com/store/apps/details?id=com.foldhueabc...
```
- 系列名：含 "APK" 的行，取整行去掉 emoji 前缀 → `FF345-APK-24`
- 链接：`📱APP链接：` 后的 URL

### 类型2：首行含系列名（"🔥" 前缀）
```
🔥神包上线：7274-apk-包37
🔥APP链接：https://play.google.com/store/apps/details?id=com.museweaveqwert...
```
- 系列名：`🔥神包上线：` 后的内容 → `7274-apk-包37`
- 链接：`🔥APP链接：` 后的 URL

### 类型3：链接在前
```
https://play.google.com/store/apps/details?id=com.Edgedod.game
应用名：Edge dod-27包
包名 com.Edgedod.game
```
- 系列名：`应用名：` 后的内容 → `Edge dod-27包`

### 类型4：名称+链接
```
谷歌11 名称：NexaPulse
https://play.google.com/store/apps/details?id=com.nexapulsezfp...
```
- 系列名：`名称：` 后的内容 → `NexaPulse`

### 类型5：纯链接
```
https://play.google.com/store/apps/details?id=com.vibe.arc3&...
```
- 系列名 = 包名（从链接提取）→ `com.vibe.arc3`

> **注意**：系列名提取规则可能随数据格式变化而调整。实现时把解析逻辑抽成独立函数，方便后续修改。

### 解析优先级
1. 提取所有 Google Play 链接
2. 每个链接找到所属的文本块（按空行/时间戳分隔）
3. 按类型1→2→3→4→5 尝试匹配系列名
4. 包名始终从链接提取

### 暂停/隐藏逻辑

- **产品暂停**：产品从主列表隐藏，仅「暂停产品」页面可见。产品下所有包跟随隐藏。
- **包暂停**（产品未停）：包行标红 + 系列名后显示 `⏸暂停` 标签。产品仍正常展示。
- **恢复**：再次点击 ⏸ 按钮取消暂停。
- **默认**：新建产品/包均为正常状态。

## 8. Excel

- **导入**：直接上传 .xlsx 文件，按表头自动匹配列，无需下拉预设
- **导出**：先按产品分组展开，产品名/KPI/地区只在每组首行显示
